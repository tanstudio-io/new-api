# V2 更新日志 - DB卡密充值系统

## 更新日期
2026-06-27

---

## 一、本次更新概述

将原有的微信支付/在线支付系统，改造为 **DB卡密（激活码）充值系统**。

### 核心变更
- **移除**：复杂的在线支付流程（自定义金额、多种支付方式选择）
- **新增**：快速充值按钮 + 激活码兑换模式
- **新增**：管理员可在后台配置充值链接，无需修改代码

### 用户充值流程（新）
```
用户点击充值按钮 (1￥/5￥/10￥...)
        ↓
跳转到9.Plus商品页面
        ↓
完成支付，获得激活码
        ↓
返回钱包页面，输入激活码
        ↓
点击"兑换"，额度到账
```

## 二、数据库变更

### 新增表：`recharge_links`

充值链接配置表，用于存储不同金额对应的外部购买链接。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键，自增 |
| `amount` | INT | 充值金额（单位：分），如 100 代表 1 元 |
| `amount_label` | VARCHAR(50) | 显示标签，如 "1￥" |
| `link` | TEXT | 充值跳转链接（如9.Plus商品链接） |
| `sort_order` | INT | 排序顺序（升序） |
| `enabled` | BOOLEAN | 是否启用 |
| `created_at` | BIGINT | 创建时间 |
| `updated_at` | BIGINT | 更新时间 |

**索引**：`amount` 字段有唯一索引

### 默认数据

系统首次启动时会自动创建 7 条默认配置（链接为空，需管理员填写）：

| amount | amount_label | sort_order |
|--------|--------------|------------|
| 100 | 1￥ | 1 |
| 500 | 5￥ | 2 |
| 1000 | 10￥ | 3 |
| 2000 | 20￥ | 4 |
| 5000 | 50￥ | 5 |
| 10000 | 100￥ | 6 |
| 20000 | 200￥ | 7 |

## 三、API 接口文档

### 3.1 用户接口

#### 获取启用的充值链接

**请求**
```
GET /api/user/recharge-links
Authorization: Bearer <token>
```

**响应**
```json
{
  "success": true,
  "message": "",
  "data": [
    {
      "id": 1,
      "amount": 100,
      "amount_label": "1￥",
      "link": "https://9.plus/shop/tanstudio/product-1",
      "sort_order": 1,
      "enabled": true
    },
    {
      "id": 2,
      "amount": 500,
      "amount_label": "5￥",
      "link": "https://9.plus/shop/tanstudio/product-2",
      "sort_order": 2,
      "enabled": true
    }
  ]
}
```

### 3.2 管理员接口

#### 获取所有充值链接（含禁用的）

**请求**
```
GET /api/recharge-link/
Authorization: Bearer <admin-token>
```

#### 创建充值链接

**请求**
```
POST /api/recharge-link/
Authorization: Bearer <admin-token>
Content-Type: application/json

{
  "amount": 3000,
  "amount_label": "30￥",
  "link": "https://9.plus/shop/tanstudio/product-new",
  "sort_order": 35,
  "enabled": true
}
```

**响应**
```json
{
  "success": true,
  "message": "创建成功",
  "data": {
    "id": 8,
    "amount": 3000,
    "amount_label": "30￥",
    "link": "https://9.plus/shop/tanstudio/product-new",
    "sort_order": 35,
    "enabled": true
  }
}
```

#### 更新充值链接

**请求**
```
PUT /api/recharge-link/:id
Authorization: Bearer <admin-token>
Content-Type: application/json

{
  "amount": 100,
  "amount_label": "1￥",
  "link": "https://9.plus/shop/tanstudio/updated-link",
  "sort_order": 1,
  "enabled": true
}
```

#### 删除充值链接

**请求**
```
DELETE /api/recharge-link/:id
Authorization: Bearer <admin-token>
```

#### 批量更新充值链接

**请求**
```
POST /api/recharge-link/batch
Authorization: Bearer <admin-token>
Content-Type: application/json

[
  {
    "id": 1,
    "amount": 100,
    "amount_label": "1￥",
    "link": "https://9.plus/shop/tanstudio/product-1",
    "sort_order": 1,
    "enabled": true
  },
  {
    "id": 0,
    "amount": 3000,
    "amount_label": "30￥",
    "link": "https://9.plus/shop/tanstudio/product-new",
    "sort_order": 35,
    "enabled": true
  }
]
```

**说明**：
- `id > 0`：更新已有记录
- `id = 0`：创建新记录

## 四、前端变更

### 4.1 钱包页面

**移除的内容**：
- 自定义金额输入框
- 支付方式选择（Stripe/Epay/Waffo等）
- 金额预设选择（带折扣计算）

**新增的内容**：
- **快速充值区域**：显示7个充值按钮（1￥/5￥/10￥/20￥/50￥/100￥/200￥）
  - 点击后调用 `window.open()` 跳转到对应链接
  - 如果链接未配置，显示错误提示
- **保留**：激活码（兑换码）输入区域

### 4.2 管理后台

**新增页面**：系统设置 → 计费 → 充值链接

功能：
- 查看所有充值链接配置
- 添加新的充值金额和链接
- 编辑已有链接
- 删除链接
- 启用/禁用链接
- 调整排序顺序

## 五、配置指南

### 5.1 管理员配置充值链接

1. 登录管理后台
2. 进入 **系统设置** → **计费** → **充值链接**
3. 为每个充值金额填写对应的9.Plus商品链接
4. 点击 **保存**

### 5.2 示例配置

假设你的9.Plus店铺有以下商品：

| 商品 | 价格 | 9.Plus链接 |
|------|------|-----------|
| 1元充值卡 | ¥1 | https://9.plus/shop/tanstudio/1yuan |
| 5元充值卡 | ¥5 | https://9.plus/shop/tanstudio/5yuan |
| 10元充值卡 | ¥10 | https://9.plus/shop/tanstudio/10yuan |

则在后台配置：

```json
[
  {"amount": 100, "amount_label": "1￥", "link": "https://9.plus/shop/tanstudio/1yuan", "sort_order": 1, "enabled": true},
  {"amount": 500, "amount_label": "5￥", "link": "https://9.plus/shop/tanstudio/5yuan", "sort_order": 2, "enabled": true},
  {"amount": 1000, "amount_label": "10￥", "link": "https://9.plus/shop/tanstudio/10yuan", "sort_order": 3, "enabled": true}
]
```

## 七、注意事项

1. **金额单位**：数据库中 `amount` 字段单位为 **分**（不是元）
   - 1元 = 100
   - 5元 = 500
   - 100元 = 10000

2. **链接配置**：新安装的系统默认链接为空，需要管理员手动填写

3. **兼容性**：原有的兑换码（Redemption）功能完全保留，与新的充值链接系统并存

4. **安全性**：充值链接仅管理员可配置，普通用户只能查看和使用启用的链接
